# .instructions.md — Quick Start for VS Code Agent

These instructions guide the VS Code agent when working on this project.

## Instant Start

**Read before any work:**
1. CLAUDE.md (workspace guidelines)
2. Agent.md (coding conventions)
3. Git log (recent patterns): `git log --oneline -10`

## Default Behaviors

### Task Intake

```
📌 Always ask for:
- File path(s) affected
- Expected behavior
- Current behavior (error message)
- What you've tried
```

### Before Writing Code

1. Read existing similar code
2. Check Agent.md for patterns
3. Run type check: `npm run build` (frontend) or `cargo check` (backend)
4. Test after changes

### Response Format

Keep it **terse**:
- Lead with: what changed
- One sentence explanation
- File path:line number for context
- No lengthy summaries

Example:
```
✅ Fixed split operation validation in AddOperationModal.tsx:67-89
Updated form submission to check total !== 0 before split.
```

## Quick Patterns

### Add React Component

```
1. Check frontend/src/components/[feature]/ exists
2. Use pattern from similar component
3. Add props interface
4. Use React Hook Form for forms
5. Use MUI for styling
6. Test in browser
```

### Add Custom Hook

```
1. File: frontend/src/hooks/use[Name].ts
2. Use React Query for data fetching
3. Return { query/mutation, ... }
4. Handle errors with notifier
5. Invalidate queries on success
```

### Add Backend Endpoint

```
1. Add model in backend/src/models.rs
2. Add handler in backend/src/handlers/[domain].rs
3. Add route in backend/src/routes.rs
4. Use SQLx for database
5. Return Result<Json<T>, ApiError>
6. Test with cargo test
```

## Emergency Patterns

### "Types don't compile"

→ Run `npm run build` — fix errors listed, not errors you assume

### "Feature doesn't work"

→ Check:
- Browser console (F12)
- Network tab (API responses)
- React Query DevTools (cache state)

### "Don't know what to do"

→ Do this order:
1. `git log -p -- <file>` — how was this done before?
2. `grep -r "pattern" src/` — find similar code
3. Check Agent.md — is there a pattern for this?
4. Ask user if still unclear

## File Location Shortcuts

```
React components       → frontend/src/components/[feature]/
Custom hooks           → frontend/src/hooks/
API calls              → frontend/src/lib/api.ts
Forms                  → use React Hook Form + Controller
Backend handlers       → backend/src/handlers/
Database models        → backend/src/models.rs

```

## Token Savers

❌ **Waste tokens:**
- Long explanations
- Explaining existing code
- Multiple back-and-forth confirmations

✅ **Save tokens:**
- Specific file paths
- Line numbers
- Git context
- Copy-paste error messages
- One-sentence summaries

## When Stuck

1. Check CLAUDE.md section "🆘 When Stuck"
2. Run `git log -p -S "keyword" -- "*.tsx"`
3. Ask for more context (copy error message)
4. Suggest 2-3 approaches, user picks one

## Common Issues & Fixes

| Problem | Solution |
|---------|----------|
| Form not validating | Check: FormProvider + Controller, Zod schema |
| Mutation doesn't update | Invalidate query: `qc.invalidateQueries({queryKey:['x']})` |
| Type errors | Run `npm run build` - fix listed errors |
| Backend error | Check `cargo test` + database migrations |
| i18n broken | Use `useTranslation()` + import keys from locale files |

## Must-Know Commands

```bash
# Frontend
npm run dev         # Start dev server
npm run build       # Type check + build
npm test            # Run tests

# Backend
cargo run           # Start server
cargo test          # Run tests
cargo fmt           # Format code

# Git
git log -p -- <file>      # See what changed
git diff main              # What changed since main
git stash                  # Save work temporarily
```

## Performance Tips

- TypeScript strict mode catches bugs early
- React Query caches automatically
- MUI components are heavy — use selectively
- BigDecimal for money (never floats)
- SQLx prevents SQL injection
